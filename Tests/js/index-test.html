My Test Framework

This Test Framework is self build for light weight javascript projects.

Coding Guidelines

  • This Framework can onyl used when certain codig guidelines are followed
  • 1) Use of Dictionaries for Function Parameters: Use a representer 'config'
  • 2) Use of Dictionaries for Function Parameters: Use a representer 'config'

Use the framework:

  • Insert the script src="../../Basis/js/Basis All_.js" /script' to your index html. the test class uses some functions from the basis js module
  • Insert the script src="../../Tests/js/test_Class.js" /script' to your index html
  • Init the test class via 'let myTest = new clsTest()'
  • Run/Call each test case and provide the test class obbject (= 'myTest' in this example) to the test as paramter.
  • At the end you can print the test results using 'myTest.PrintResult(divID)', where 'divID' is the id of the DOM element where the results are written to.

Write Test Cases

  • Each test case gets the clsTest() object (='myTest') provided as a parameter, e. g. 'function test_NoBlanksInList(myTest)'
  • Within the test you can use one of the four class test functions 1) Equal, 2) IsTrue, 3) IsFalse, 4) Assertion
  • The class test functions work with all sorts of data types, including lists/arrays and dictionaries/objects
  • The class test functions require also the current ego name of the test case (actually current javascript function) name that is running. Yuu get this information via 'arguments.callee.name'

Optional Features

  • You can print all functions that have been called, e. g. to get a coverage of functions. The 'TraceFunctionCalls = []' serves as a collector for all functons being called during the test runs
  • Therfore you have to insert 'TraceFunctionCalls.pushX(arguments.callee.name)' to every function at the very beginning. This is not recommended for productive code. But you can Copy Paste ypur codefile(s) and insert the line and then test this modified code
